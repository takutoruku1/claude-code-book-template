## 1. チームの存在意義

### なぜこのチームが必要か

シナリオ制作チーム（世界観担当・演出担当・ビジュアル担当）は「作る視点」から機能する。各担当が自分の領域を正確に、高品質に完成させることに集中している。それ自体は正しい。

しかし、すべての部品が正確に動いていても、**プレイヤーが「面白い」と感じるかどうか**は別の問題だ。

具体的に言えば:
- セリフは正確に書かれているが、読んでいて感情が動かない
- 演出タイミングは設計書通りだが、緊張感が生まれていない
- エンディングは複数存在するが、2周目をやろうという気にならない

これは「作った人間には見えにくい問題」だ。作り手は意図を知っているため、プレイヤーが感じる「わからなさ」「退屈さ」「後悔のなさ」を体験できない。

**面白さ保証チームは「体験する視点」の専任担当として機能する。**

シナリオ制作チームと面白さ保証チームの違いを一言で言えば:

| 軸 | シナリオ制作チーム | 面白さ保証チーム |
|---|---|---|
| 視点 | 作る視点（設計の正確さ） | 体験する視点（感情の動き） |
| 問い | 「正しく実装されているか」 | 「面白いか」 |
| 動くタイミング | Phase 1〜3（制作フェーズ） | Phase B（完成後レビューフェーズ） |
| 出力 | セリフ・演出・ビジュアル仕様 | 評価レポート・改善指摘 |

このゲーム「バズったー」の面白さの核は以下の2点だ。

1. **「バレたら終わり」の緊張感** — 湊がクライアントのなりすまし運用をしている事実が、常にプレイヤーの背景に漂っている。この緊張感が薄れると全体のトーンが崩壊する。
2. **依頼主への感情移入** — みどり・朔・誠司・花蓮それぞれが「この人のためにやってやりたい」と思えることが選択肢の重みを生む。感情移入が弱いと選択肢が「どちらでもいい」になる。

面白さ保証チームは、この2点が実際に機能しているかを継続的に検証する。

## 2. ① プレイヤー心理担当 — 詳細チェックリスト

### 担当範囲

プレイヤーが「この人のために選びたい」「このシーンが怖い」「この結末は辛い」と感じるかどうかを担当する。感情移入・緊張感・選択肢の重みの三軸を評価する。

---

### 感情移入チェックリスト（10項目）

各項目を A（強く達成）/ B（概ね達成）/ C（弱い）/ D（未達成）で評価する。

| # | チェック項目 | 評価 | コメント欄 |
|---|---|---|---|
| 1 | キャラクターの「困り方」に具体性があるか（抽象的な悩みではなく、このシーンこの状況固有の苦しさか） | — | |
| 2 | 依頼主が「湊に頼む理由」が感情的に納得できるか（スペックではなく、信頼の根拠が見えるか） | — | |
| 3 | 湊のモノローグが「翻訳者としての内面」を見せているか（事務的な解説になっていないか） | — | |
| 4 | 選択肢の前後で「何かを失うかもしれない」感覚があるか | — | |
| 5 | クライアントのSNSリアクションが「その人らしい反応」になっているか（汎用的な喜び方ではないか） | — | |
| 6 | 「バレたら終わり」の圧力がシーン内で具体的に想起されるか（抽象的な不安ではなく、誰にどうバレるかが見えるか） | — | |
| 7 | キャラクター同士の関係性が選択肢に影響を与えているか（誰でも同じ選択になっていないか） | — | |
| 8 | セリフの語尾・口調がキャラクター固有のものになっているか（他のキャラに置き換えても成立しないか） | — | |
| 9 | エモーショナルな場面で「少し間を置きたくなる」設計になっているか（情報が多すぎて感情が追いついていないか） | — | |
| 10 | このシーンの後、次のシーンに進みたいと思えるか（強制感ではなく、続きを知りたいという欲求が生まれるか） | — | |

---

### 離脱リスク判定基準

以下のいずれかが発生していると「離脱リスク高」と判定する。

**即時離脱リスク（Dアラート）:**
- 読み飛ばしたくなるほど長い説明セリフが連続している（3行超の説明が2回以上連続）
- 選択肢が「どちらでも同じ結果になる」と予見できるほど差異がない
- BGMや演出の変化なしに感情ピーク場面に入っている

**中期離脱リスク（Cアラート）:**
- 2シーン以上、キャラクターの「困り方」に具体性がない状態が続いている
- 同一ルート内で同じトーン（緊張 or 和らぎ）が4シーン以上継続している
- 湊のモノローグがすべて状況説明になっており感情が見えないシーンが続く

---

### 選択肢の迷い度評価方法

目標: プレイヤーが「3秒以上迷う選択肢」を全選択肢の70%以上にすること。

**評価基準:**

| グレード | 定義 | 具体的状態 |
|---|---|---|
| A（強く迷う） | 両方の選択肢に「捨てたくないもの」がある | どちらかを選ぶと何かを失う構造になっている |
| B（迷う） | 一方に明確な優位性があるが、もう一方にも捨てがたい要素がある | 「普通ならBだが、このキャラにはAかもしれない」と思える |
| C（迷わない） | 正解が自明 | 一方が明らかに良い選択肢、もう一方がリスクのみ |
| D（意味がない） | どちらを選んでも実質同じ | ルート分岐も感情的差異もない |

**判定手順:**
1. 選択肢の文言を「このキャラを知らない第三者に見せた場合」どちらが正解か判定する
2. 即座に正解が分かれば C 以下
3. 「このキャラならどちらか」を考える必要があれば B 以上
4. キャラを知っていても迷うなら A

## 3. ② テンションカーブ担当 — 詳細チェックリスト

### 担当範囲

1ルート全体・1シーン内の緊張感の起伏を数値化し、グラフとして可視化する。平坦化・溜め不足・ピーク過多のパターンを検出し、修正指示を出す。

---

### テンションスコア定義表

| スコア | 意味 | 典型的な状態 |
|---|---|---|
| 0 | 無力感・絶望 | 取り返しのつかない失敗が確定した直後、沈黙、暗転 |
| 1 | 深い落ち込み | エンディングの余韻（悪エンド後）、喪失感が続いている |
| 2 | 憂鬱・不安 | 伏線の匂い、嫌な予感、何かがおかしいと気づき始める |
| 3 | 静かな緊張 | 何かが起きそうな日常、準備フェーズ、穏やかだが気が抜けない |
| 4 | 日常・関係構築 | キャラクターとの関係が深まる会話、温かみのあるやり取り |
| 5 | 普通の高揚 | 新しい依頼がきた、素材が揃った、小さな成功 |
| 6 | 期待・ワクワク | 投稿が完成した、送信する直前、何かが動き始める予感 |
| 7 | 強い緊張 | 送信後のリアクション待ち、誰かに見られているかもしれない |
| 8 | 危機的緊張 | バレるかもしれない状況が具体化、リアルタイムで追い詰められる |
| 9 | 恐怖・限界 | 湊の過去が絡む、重大な判断を迫られる、感情が爆発する直前 |
| 10 | 最高潮 | エンディング直前の決断、取り返しのつかない瞬間 |

### 平坦ゾーン判定基準と対処パターン

**平坦ゾーンの定義:**

以下のいずれかを満たす連続シーンを「平坦ゾーン」と判定する。

| 種別 | 条件 |
|---|---|
| 低平坦 | スコア4以下が3シーン以上連続 |
| 中平坦 | スコア4〜6の範囲に5シーン以上留まる |
| 高平坦 | スコア7以上が4シーン以上連続（ピークの形骸化） |

**対処パターン:**

| 平坦種別 | 対処パターン | 具体例 |
|---|---|---|
| 低平坦 | 「嫌な予感」シーンの挿入 | 伏線的な一言を誰かが言う、過去の失敗を湊が思い出す |
| 低平坦 | 関係値の「揺らぎ」演出 | クライアントが急に無口になる、既読無視 |
| 中平坦 | 小さなピークの設計 | 通知音、いいね数の急増、DM着信 |
| 中平坦 | 選択肢の重みを増す | 次の選択に「バレるかも」のリスクを明示的に追加 |
| 高平坦 | テンション落とし込み | 静かな場面、キャラクターが弱さを見せる瞬間を入れる |

---

### 「溜め」の設計チェック項目

ピーク（スコア9〜10）の前に必ず「溜め」設計があるかを確認する。

| チェック項目 | 基準 |
|---|---|
| ピーク前2シーン以上が「溜め」構造になっているか | ピーク直前のシーンだけで溜めているのは不十分 |
| 溜めのシーンで「何かが起きそう」という予感が作られているか | スコア6〜7の範囲に収まっているか |
| 溜めシーン内に「まだ大丈夫」という安心要素も同居しているか | 緊張一辺倒は溜めではなく別ジャンルのプレッシャー |
| ピーク場面の直前に「一瞬の静けさ」があるか | 演出上のブレスの確認（wait演出、BGMフェードなど） |
| ピーク後に「余韻」シーンが設計されているか | スコアが急に日常値（4〜5）に戻っていないか |

## 4. ③ リプレイ価値担当 — 詳細チェックリスト

### 担当範囲

1周目クリア後に「もう1周やりたい」と思えるかを担当する。伏線・発見・感情再解釈・選択後悔の4軸でリプレイフックを評価する。

---

### リプレイフック分類表

| 分類 | 定義 | このゲームでの例 |
|---|---|---|
| **伏線型** | 1周目では意味がわからなかったセリフ・演出が、2周目で意味を持つ | 花蓮の「あの人みたいね」が1周目では謎で、2周目で湊の過去と繋がる |
| **発見型** | 1周目では見えなかった選択肢・分岐・イベントが2周目で見える | 誠司ルートで特定の選択をしていた場合だけ出る隠しセリフがある |
| **感情再解釈型** | 別ルートをクリアした後、最初のルートを見直すと感情の意味が変わる | みどりルート後に朔ルートをやると、みどりの言動の意味が変わる |
| **選択後悔型** | 選んだ選択肢と逆の選択をしたらどうなったか気になる | 「あそこで朔を庇わなかったらどうなっていたか」 |

**評価基準:** 4分類のうち3つ以上が「B以上」の強度で機能していること。

---

### ルート間参照チェックシート

4ルートのキャラクター間で「知っていると深まる関係性」を確認する。

| | みどり | 朔 | 誠司 | 花蓮 |
|---|---|---|---|---|
| **みどり** | — | みどりが朔の過去を知っている描写があるか | みどりと誠司に共通する「失敗体験」が示唆されているか | みどりルートの選択が花蓮ルートの伏線になっているか |
| **朔** | 朔はみどりをどう見ているか描写されているか | — | 朔と誠司の間に「競争関係」の匂いがあるか | 朔ルートの結末が花蓮の謎に関係するか |
| **誠司** | 誠司のSNS観がみどりと対比になっているか | 誠司が朔の存在を意識している描写があるか | — | 誠司が過去に花蓮（篠宮カレン）を知っていた可能性があるか |
| **花蓮** | 花蓮がみどりの名前を知っているか | 花蓮が朔に言及するシーンがあるか | 花蓮と誠司の接点が示唆されているか | — |

**チェック方法:** 各セルについて「あり（具体的なセリフ・演出で示されている）」「示唆あり（読者が想像できる程度）」「なし」で評価する。「なし」が4セル以上あると、ルート間連結が弱いと判定する。

---

### エンディング差別化チェック

8つのエンディングそれぞれについて以下を確認する。

| チェック項目 | 基準 |
|---|---|
| このエンディングでしか得られない「感情」があるか | 他のエンドでも似た余韻が得られるなら差別化不足 |
| エンディングのラストシーンが「画像として記憶に残る」か | 抽象的な説明エンドになっていないか |
| 「違う意味で報われる」構造になっているか | 良エンドは「ハッピー」、別の良エンドは「納得」、と意味が違うか |
| 悪エンドが「ただ辛い」ではなく「意味のある辛さ」になっているか | 悪エンドを経て次のプレイへの動機が生まれるか |
| 主人公・湊の成長または変化がエンドごとに異なる形で描かれているか | 全エンドで湊が同じ結論に達していないか |

---

### 評価出力テンプレート例

```markdown
## リプレイ価値評価 — 誠司ルート

**評価グレード**: B

### リプレイフック評価

| 分類 | 強度 | 具体的フック | 判定 |
|---|---|---|---|
| 伏線型 | A | 第2シーンの「10年前の話は今も続いている」が花蓮ルートで意味を持つ | 機能している |
| 発見型 | C | 隠し選択肢が1つしか存在しない。発見の喜びが薄い | 要強化 |
| 感情再解釈型 | B | みどりルート後に誠司の「数字で見ない」という姿勢が対比として機能する | 機能している |
| 選択後悔型 | B | 第7シーンの選択は逆を選びたくなる構造になっている | 機能している |

**達成率**: 3/4（目標達成）

### ルート間連結

- みどり↔誠司: 示唆あり（対比として機能）
- 朔↔誠司: なし（要検討）
- 花蓮↔誠司: あり（強い連結）

**懸念**: 朔ルートとの接点がほぼないため、朔→誠司の順にプレイしたプレイヤーにとって「ルートをまたいだ発見」が薄い

### エンディング差別化

- 真エンド「翻訳者」: 評価A — 湊の贖罪テーマが完結する独自の感情を持つ
- ノーマルエンド「代行の限界」: 評価B — 余韻はあるが「翻訳者」エンドと感情が近い
- バッドエンド「繰り返し」: 評価B — 意味のある辛さとして機能している

### 主要指摘

1. **[推奨]** 発見型フックを1〜2箇所追加する（隠し選択肢または隠し演出）
2. **[推奨]** 朔ルートとの連結点を1箇所設計する
3. **[良点]** 花蓮との連結は強く、クロスルートのリプレイ動機として十分機能している
```

## 5. 「面白くない」のパターン集（このゲーム固有）

このゲームで実際に発生しやすい問題パターンを記録する。制作チームへの指摘・修正フィードバックに使用すること。

---

### パターン A: 選択肢が「どちらでもいい」になる

**症状:** プレイヤーが迷わず即選択し、選択後も「逆を選べばよかった」と思わない。

**原因のパターン:**

1. **コスト差がない**: 両選択肢に「失うもの」が設計されていない。リスクがない選択肢は重みを持たない。
2. **正解が文脈から自明**: 「丁寧に書く」vs「雑に書く」など、常識的な正解が存在する選択肢。このゲームの選択肢はすべて「どちらも一定の正しさを持つ」設計であるべきだ。
3. **キャラクター固有性がない**: どのルートでも同じ選択肢が並んでいる。みどりに最適な選択肢は、朔への最適解とズレていないといけない。
4. **結果が遠すぎる**: 選択の影響が数シーン後にしか出ない場合、選択時の重みが薄れる。選択直後に小さな手触りがあることが重要。

---

### パターン B: テンション平坦化（素材選択フェーズでよく起きる）

**症状:** 投稿作成フェーズ（素材・ハッシュタグ・文章の選択）でゲームのテンポが落ち、緊張感が消える。

**原因:**

1. 素材選択が「クイズ」になっている: 正解がわかれば即選択できる。迷いがない。
2. 選択の意味がすぐわからない: 選んだ素材がどう機能するか、選択時には見えない。
3. タイム感・プレッシャーがない: 「今すぐ送らなければ」「これを見られたら」という圧力が演出されていない。
4. キャラクターが静かすぎる: 素材選択中に依頼主の「期待値」「不安」が見えないと、プレイヤーの感情が動かない。

**対処:** 素材選択フェーズ内に依頼主の一言（チャットメッセージ）を挟む。湊の内心モノローグで「バレるかもしれない」要素を明示する。

---

### パターン C: 花蓮ルート固有の問題（謎が早解けする）

**症状:** 花蓮 = 篠宮カレン（湊が過去に追い詰めた人物）であることが、設計より早い段階でプレイヤーに分かってしまう。

**原因のパターン:**

1. 名前・外見への言及が早すぎる: 第2シーン以前に「カレン」への言及がある場合、ほぼ確定してしまう。
2. 湊の反応が過剰: 花蓮登場シーンで湊が明らかに動揺していると、初見でも気づく。
3. 他ルートのキャラが花蓮に言及しすぎる: 「あの花蓮さん」のような固有名詞を使うと、花蓮ルートを後にやったとき謎が消えている。

**原則:** 花蓮の正体は「花蓮ルート第7シーン以降に確信」の設計とする。それ以前はプレイヤーが「もしかして」と思える程度の伏線にとどめる。

---

### パターン D: セリフが「説明的」になって感情移入が壊れる

**症状:** キャラクターがプレイヤーに向けて状況・感情・背景を説明するセリフを発する。読んでいて世界から引き剥がされる感覚が生じる。

**具体例（NG）:**
> 「私はSNSをやるのが怖いんです。昔、炎上して大変な思いをしたので。だから湊さんに頼んでいるんです」

**具体例（OK）:**
> 「…あのアカウント、もうないんです」（沈黙）「消したんですよね、自分で。当時」

**原則:**
- 感情・背景は「行動・沈黙・反応」で示す
- 説明は湊のモノローグに移す（キャラが自分を解説しない）
- セリフ内に「なので」「だから」で始まる因果説明が入ったら要注意

---

### パターン E: エンディングの余韻が足りない

**症状:** エンディングシーンが終わった後、感情が残らない。「で、終わり？」と感じる。

**原因のパターン:**

1. **結論を語りすぎる**: エンディングセリフが「これからの湊はこうなります」と未来を説明してしまう。余韻は「余白」から生まれる。
2. **音楽・演出が先に終わる**: BGMが先にフェードアウトし、最後の画面が無音・無演出で表示される状態は余韻を殺す。
3. **ラストカットに動きがない**: スチル1枚のまま字幕が流れるだけのエンディングは記憶に残りにくい。わずかな動き（パーティクル・フェード・揺れ）があるだけで異なる。
4. **直後にタイトルに戻る**: エンディングの直後にタイトル画面に戻ると「強制的に日常に戻される」感覚が生じる。エンドロール・静止・空白の時間を必ず確保する。

## 6. Fun Check Phase B — トリアージ判断基準

面白さ保証チームの3担当から評価が出た後、メインが以下の基準でトリアージを行う。

---

### 重複指摘の扱い

**3担当中2名以上が同じ問題を指摘した場合:**
- 自動的に「優先対処」とする
- 修正指示をシナリオチームへ差し戻す（後述の形式で）
- 問題の記述は「最も具体的に書かれたもの」を採用し、他は参照として添付する

**3担当全員が同じ問題を指摘した場合:**
- 「緊急対処（リリースブロッカー）」として扱う
- 当該シーンの実装を一時停止し、修正完了後に再評価する

---

### 単独指摘のトリアージ基準

3担当のうち1名だけの指摘は、以下の基準で優先度を判定する。

| 条件 | 判定 |
|---|---|
| プレイヤー心理担当の「感情移入D」項目への指摘 | 優先対処 |
| テンションカーブ担当の「平坦ゾーン3連続」への指摘 | 優先対処 |
| リプレイ価値担当の「伏線型フック機能不全」への指摘 | 優先対処 |
| 上記以外の単独指摘（スコアC程度） | 推奨対処（次スプリント） |
| 改善提案（「〜するともっと良い」系） | 任意対処（余力があれば） |

---

### 「修正しない」判断をしていい条件

以下をすべて満たす場合、メインは「修正しない」と判断してよい。

1. 単独指摘（2名以上の重複ではない）
2. 修正によって別の担当の評価が下がるリスクがある（トレードオフが明確）
3. 指摘された問題が「好みの問題」の範囲に収まっている（客観的な面白さ阻害要因ではない）
4. 当該シーンの全体評価がBまたはA以上（致命的ではない）

この判断をした場合、メインは「修正見送り・理由：〇〇」を記録する。記録なしの見送りは不可。

---

### シナリオチームへの差し戻し形式

```markdown
## 差し戻し指示 — [対象シーン名]

**差し戻し理由**: [重複指摘 / 優先対処 / 緊急対処]
**対象担当**: [世界観担当 / 演出担当 / ビジュアル担当] のいずれか（または複数）

### 修正が必要な問題

**問題1**: [問題の具体的な記述]
- 出典指摘: [担当名] の評価レポート参照
- 修正方向: [具体的な修正の方向性。「こうせよ」ではなく「この効果を達成せよ」で記述する]

**問題2**: [問題の具体的な記述]
- 出典指摘: [担当名] の評価レポート参照
- 修正方向: [具体的な修正の方向性]

### 制約条件

- 変えてよいもの: [セリフ文言 / 演出タイミング / 選択肢設計 など]
- 変えてはいけないもの: [このシーンで必達の要件 / 他シーンとの整合性が必要な部分]

### 再評価タイミング

修正後、面白さ保証チームの[プレイヤー心理担当 / テンションカーブ担当 / リプレイ価値担当]が再評価する。
```

## 7. 評価サンプル（架空例）

### 対象: みどりルート 第3シーン（架空）

**シーン概要（架空）:** 湊がみどりの代わりに書いたエッセイ投稿が公開され、みどりの友人からコメントが殺到する。感情的なコメントに混じって「本当にみどりが書いたの？」という一言が届く。

---

### ① プレイヤー心理担当 — 評価

**評価グレード: C**

**主な指摘:**

1. **[重大]** 「本当にみどりが書いたの？」のコメントへの湊の反応が「少し焦る」という表現にとどまっており、「バレたら終わり」の緊張感が薄い。このシーンは緊張のピークになるべきだが、感情移入チェックの項目6（バレたらの圧力）がDと評価された。
2. **[推奨]** 選択肢「無視する」vs「湊として返信する」の2択だが、「無視する」を選ぶリスクが明示されておらず、正解が自明になっている。迷い度C判定。
3. **[良点]** みどりのリアクション（嬉しさと不安が混在するメッセージ）はキャラクター固有性が高く、依頼主への感情移入を補っている。

---

### ② テンションカーブ担当 — 評価

**評価グレード: C**

**主な指摘:**

1. **[重大]** 前シーン（第2シーン）のテンションスコアが4（日常）から第3シーンが6（期待）と、「バレかけ」の場面なのにスコアが低すぎる。本来このシーンは7〜8であるべき。溜めが不足している。
2. **[推奨]** 「本当にみどりが書いたの？」のコメント着信の演出が通常の通知と同じトーンになっており、テンションの変化点が音・視覚的に強調されていない。演出的な溜めが必要。
3. **[良点]** シーン後半のBGMフェードは設計通りで、余韻の処理は適切。

---

### ③ リプレイ価値担当 — 評価

**評価グレード: B**

**主な指摘:**

1. **[推奨]** 「本当にみどりが書いたの？」のコメントを書いたアカウントが花蓮ルートで登場するキャラクターである可能性が示唆できる。現状は汎用的なモブとして処理されており、伏線型フックの機会を逃している。
2. **[推奨]** 「無視する」選択をした場合の分岐が短く、逆の選択への後悔を生みにくい。
3. **[良点]** みどりの「この投稿、なんか私じゃないみたいで怖い」というセリフは、2周目に花蓮ルートをクリアした後で意味が変わる。感情再解釈型フックとして機能している。

---

### メインによるトリアージ結果

| 指摘 | 担当数 | 判定 | 対処 |
|---|---|---|---|
| 緊張感が薄い（バレたら感が弱い） | 2名（心理・テンション） | 優先対処 | 世界観担当・演出担当に差し戻し |
| 選択肢「無視する」のリスク不明確 | 2名（心理・リプレイ） | 優先対処 | 世界観担当に差し戻し |
| 通知演出の強調不足 | 1名（テンション） | 優先対処（テンションDアラートに相当） | 演出担当に差し戻し |
| 花蓮との伏線機会 | 1名（リプレイ） | 推奨対処（次スプリント） | 記録のみ、今回は対処しない |

**トリアージ結論:**
- 緊張感・選択肢設計・通知演出の3点を優先修正とし、シナリオ制作チームに差し戻す
- 花蓮との伏線機会は、花蓮ルート設計フェーズで検討する課題として記録する
- 全体評価: **C**（優先修正後に再評価を行う）
