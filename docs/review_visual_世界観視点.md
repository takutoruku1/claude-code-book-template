> **レビュー対象**: `docs/visual_spec.md` v1.0

## 総合評価: B

仕様書全体として、ゲームのトーンと視覚的方向性の把握は正確である。特にグリッチエフェクトの3段階設計（mild→medium→severe）はシナリオの感情弧に概ね沿っており、「静かな恐怖の累積」という世界観的要件を視覚言語として翻訳しようとする意図は評価できる。ただし、いくつかの重大な齟齬があり、シナリオが設計した感情弧を毀損するリスクがある。

---

## 重大な問題点

### 問題①: グリッチがルートをまたいで「花蓮色」に固定されている

`docs/plot_steinsgate.md §4（感情弧マップ）` によれば、みどりルート・朔ルート・誠司ルートのそれぞれに固有の「恐怖の質」がある。みどりルートは「翻訳」という言葉が微かに引っかかる程度であり、朔ルートでは「名前」、誠司ルートでは「居場所の喪失」という、互いに異なるテクスチャの不安が蓄積される。

しかし `visual_spec.md §1-4` および `§2` のミステリーフェーズ演出は、ルートを問わず `rgba(200, 120, 144, 0.xx)` ——すなわち **花蓮ルートのダークローズ** で統一されている。Phase 2 のウィンドウ枠の赤み、Phase 3 のタイトルバー「傷」アニメーション、グリッチオーバーレイの `prefers-reduced-motion` 代替静的カラーすべてが同じ色だ。

これは世界観上、致命的な問題である。みどりルートや朔ルートをプレイ中に「花蓮色の違和感」が漂うと、プレイヤーはあらかじめ「最終的に花蓮が重要なのだ」と感知してしまう。シナリオは、各ルートの恐怖をそのルートの文脈で積み上げ、花蓮ルートで初めて収束させる構造だ（シュタゲ的構造）。ミステリーフェーズの色が最初から花蓮ルートのものであれば、「収束の驚き」が失われる。

**改善案**: ミステリーフェーズのグリッチカラーは、現在プレイ中のルートの `--route-accent` を参照するよう設計すること。具体的には `rgba(200, 120, 144, 0.xx)` をハードコードするのではなく、`rgba(var(--route-accent-rgb), 0.xx)` または CSS変数 `--mystery-crack-color` を定義してルートごとに上書きする構造にする。花蓮ルートのみ最終的にダークローズが使われることで、「この色が全伏線の答えだった」という視覚的回収になりうる。

加えて、`plot_steinsgate.md §1` が示す伏線密度（みどり0〜10% / 朔30% / 誠司70% / 花蓮100%）に対応して、ルートごとのグリッチ上限フェーズを CSS で制約すること。少なくとも `body[data-route="midori"]` では `--mystery-glitch-opacity: 0` を強制し、みどりルートが「安心ゾーン」として機能することを保障する必要がある。

---

### 問題②: 「翻訳」というテーマがビジュアルに存在しない

`plot_steinsgate.md §1（みどりルートの伏線①）` および `§5（既読エンドの設計意図）` に記されているとおり、「翻訳」はこのゲームの中心テーマである。みどりルートで「翻訳が上手くいった」という湊の誇りとして種が蒔かれ、花蓮ルートの贖罪エンドで「翻訳、ありがとうございました」として収束する。このモチーフはセリフ・ナラティブだけでなく、**視覚的に対応するメタファーが必要**だ。

`visual_spec.md` 全体を通して「翻訳」という概念に対応するビジュアル要素が存在しない。具体的には以下の欠落がある:

1. **湊の「透明性」を表すビジュアル概念がない**: 翻訳者として「自分が消える」という湊の職業的アイデンティティをUIで表す演出（例: 主人公バブルの透明度変化、モノローグ時の色の引き算）が設計されていない。

2. **フラッシュバック3の「2枚並列演出」のCSSが未定義**: `plot_steinsgate.md §3「【フラッシュバック3の詳細設計】」` は「Phase 1 のテキストを背景に薄く再掲し、その上に k_memo カードが重なる」と明示しているが、この二層テキストレイヤー演出の具体的CSSが `visual_spec.md` にない。これはシナリオの視覚的クライマックスでありながら未定義のままだ。

3. **沈黙エンドの「差出人不明のメッセージ」に専用演出がない**: `plot_steinsgate.md §5` で沈黙エンドのラストメッセージ（`——カ`）は最も余韻の深い瞬間として設計されているが、`visual_spec.md §4-2` の沈黙エンド仕様は `background: #080808`（漆黒）のみで、テキストが画面に浮かび上がる演出が定義されていない。

**改善案**:
- 湊モノローグバブル専用スタイルを追加定義する（通常チャットバブルより透明度を高めた表示、`opacity: 0.85` + `font-style: italic` 等）。
- 沈黙エンド用の「文字がゆっくり浮かび上がる」テキスト演出CSSを定義する（例: `opacity: 0 → 0.6` の 4 秒 `ease-in` で完全には見えないまま留まる）。
- フラッシュバック3の二層テキスト演出（過去テキストの `::before` 透過レイヤー + 現在カードの重ね表示）のCSSを追加すること。

---

### 問題③: 花蓮クライマックスの「暴走ビジュアル」がカタルシスより先行しすぎる

`visual_spec.md §4-1` の「バズ数字が暴走する」演出は、`karen_mystery_choice` の選択肢表示**直前**に発動するとされている。しかし `plot_steinsgate.md §3（「【花蓮が語る場面】karen_mystery_choice の直前チャット」）` によれば、クライマックス直前は花蓮が静かに「3年前、SNSで誰かに書かれた記事で、姉が消えました」と語る**最も静寂な場面**だ。

バズ数値の暴走・いいねアイコンの「狂ったような点灯」・ハッシュタグの赤い滲みといった激しい演出が、花蓮の静かな告白と同時または直前に起動していると、感情の方向性が衝突する。シナリオの感情設計は「外側（SNS）の騒音」と「内側（花蓮との対話）の静寂」を対比させることで緊張を生んでいるが、視覚演出が「外側の騒音」を全面に押し出すことで、対話の重みが失われる。

さらに `§4-2` の `.karen-resolve-fade` アニメーションは「70%で温かい色調へ」「収束後に金色 `rgba(240, 208, 128, 0.08)` が入る」という設計だが、シナリオの贖罪エンドは「赦しとも断罪とも取れる保留」で終わる（`plot_steinsgate.md §5`）。視覚が「解決した」と先に語ることで、花蓮の「だから、頼みました。」という台詞の余韻が消える。

**改善案**:
- 暴走演出の発動タイミングを「フラッシュバック3発火（k_memoめくり後2500ms）直後」に前倒しし、花蓮が「湊さん？」と気遣いを見せる場面で意図的に演出を「収める」設計にすること。花蓮の告白シーンは暴走後の「静寂」として機能させることで、感情的コントラストが生まれる。
- `.karen-resolve-fade` の「温かい色調への収束」演出は贖罪エンド（`.ending-zange`）専用にスコープを絞る。既読エンド（`.ending-kidoku`）の収束は「藍色のまま落ち着く」設計にし、沈黙エンドでは一切の解決色（金・セピア・温暖色）を使わないこと。

## 改善推奨

### 推奨①: 朔ルートのグリッチを「寒色・低温方向」に専用化する

朔ルートの感情は「工場・静寂・深夜の孤独」だ。現状のミステリーフェーズ演出はルートを問わず同一の `hue-rotate` 方向（暖色化）になっている。朔ルートのPhase 2〜3では、グリッチを「ノイズの多い激しい揺れ」ではなく「低温・持続的な色温度の低下」として表現することが世界観に合う。`hue-rotate` を暖色方向ではなく寒色方向（マイナス方向）に振り、`saturate` を下げる方向のフィルターを朔ルート専用で追加することを推奨する。

### 推奨②: 誠司ルートの「ゆきわりそう」発光を「古い電球」の質感にする

`§5-3` の `.yukiwarisou-kw` のホバーエフェクトはセピアゴールドで整合しているが、発光エフェクト（`text-shadow` のglow）が「喫茶・昭和レトロ・疲労感」の世界観にやや相容れない。蛍光灯が明滅するような `animation` よりも、「古い電球が点滅する」ような不均一なちらつき（イージングに `steps()` を使った非連続な点灯）の方が誠司ルートの世界観を強化する。

### 推奨③: 花蓮の「知っていた顔」アバター（`.karen-avatar[data-mood="knowing"]`）の演出強化

`§3-2` の `.karen-knowing` は現在「脈動する赤みのグロー」として設計されているが、シナリオ上の花蓮は「依頼者として計算して動いている知性的な人物」でもある。単なる「不気味さ」ではなく「静かな強さ」を加えるため、`border-width` を微妙に増やす（2px→2.5px）変化をアニメーションに組み込み、「境界が明確になる」演出にすることを推奨する。これにより「謎めいた」から「凛とした」への移行を視覚的に予感させる。

### 推奨④: `@hanaren_log` リプライの視覚的異質感の定義

`plot_steinsgate.md §1 誠司ルート伏線④` によれば、`@hanaren_log` のリプライは「いいね: 0, リプ: 0、ほぼ投稿がないアカウント」として設計されており、反響フィードの中で意図的に「目立たないが異質」な存在として配置される。しかし `visual_spec.md §5` の反響フィード関連演出はいいね数の「暴走」や「赤染まり」にフォーカスしており、「孤立した静かなリプライ」の視覚化——つまりこのリプライが他のリプライと異なる理由が分かる微妙な差分演出——が設計されていない。`opacity: 0.7` や `font-size` を微細に小さくするだけでも、「これは違う」という直感をプレイヤーに与えられる。

## 良かった点

### 良①: 花蓮アバターの段階的変化が感情弧と整合している

`visual_spec.md §3-1` の花蓮アバター仕様（normal → unsettled → revealed）は、シナリオの感情フェーズ（平常→不穏→真実告白）と正確に対応している。特に `.karen-avatar.revealed` の「金色への昇華」（`border-color: #f0d080`）は、花蓮が「既知の上で依頼した」という真意が明かされる場面のカタルシスと整合している。ダークローズが金へと変容するこの演出は、シナリオが設計した感情の変容を視覚的に追体験させる設計として評価できる（ただし問題③で指摘した通り、エンディング別のスコープ分離は必要）。

### 良②: みどりアバターの「flashback-hint」演出がシナリオと一致している

`.midori-avatar[data-mood="flashback-hint"]` の「アバターが一瞬固まる」演出（`midori-freeze`）は、シナリオが設計した「お母さんワードへの反応」と精密に合致している。「明度低下 → 維持 → 元に戻る」という構造で、プレイヤーが「気づけるか気づけないかの境界線」に設計されている点が世界観に忠実だ。

### 良③: 「ゆきわりそう」ハイライトが誠司・花蓮ルートをまたぐ伏線装置として機能している

`§5-3` の実装は、伏線キーワードがチャットバブル内で視覚的に「光る」設計であり、一度気づいたプレイヤーが後から読み返した際に「あそこで既にあったのか」と発見できる構造になっている。`body[data-yukiwarisou="found"]` によるグローの強化も、シナリオが設計したクルー蓄積後の「既知の恐怖」と対応している。なお、花蓮ルートでも `#c8a878`（誠司カラー）が残ることは、「ゆきわりそうは誠司の世界に属する場所」というシナリオの設計意図と一致しており、意図的であれば正しい選択だ。

### 良④: 贖罪エンドのタイピング演出が湊の葛藤を体現している

`§4-2` の `.zange-typing-wrap` と `.zange-typing-cursor` による「書きかけて消す → 再入力」演出は、`plot_steinsgate.md §5` に記述された「翻訳者が、初めて翻訳しないで言葉を送る瞬間」というシナリオの核心を、UIアニメーションで直接表現している。カーソルを `--accent2`（#7c6af7）で点滅させる選択も、「これは湊自身の言葉である」という強調として機能する。シナリオ担当として、この演出は特に評価する。

### 良⑤: グリッチ強度の3段階骨格がシナリオの感情弧と概念的に合致している

mild→medium→severe の段階設計は、`plot_steinsgate.md §4` の感情弧マップ（みどり→朔→誠司→花蓮の段階的恐怖密度）と概念レベルで一致している。特に `mild` の「ほぼ気づかない程度（hue-rotate: 4deg）」という微細さは、みどりルートの「翻訳」という言葉が一瞬引っかかってすぐ流れる感覚と呼応している。問題①で指摘した「色の問題」と「ルート横断の問題」を修正すれば、この骨格はそのまま生かせる。

## まとめ

最も急ぎ対処すべきは以下の3点:

1. **ミステリーフェーズのグリッチカラーをルート変数化**（問題①）— 全ルートで花蓮色が漏れることで、シュタゲ的「収束の驚き」が失われる。最優先で修正が必要。
2. **「翻訳」テーマの視覚言語化**（問題②）— ゲームの中心テーマがビジュアルに存在しない。最低限、フラッシュバック3の二層演出定義と沈黙エンドのテキスト浮上演出を追加すべき。
3. **暴走演出と静寂告白の発動タイミング分離、および収束演出のエンディング別スコープ化**（問題③）— 演出がシナリオの感情弧を先回りすることで、「保留の余韻」が消える。

これら3点を修正することで、ビジュアル設計はシナリオの感情弧を正確に支える水準（A評価）に達する。
