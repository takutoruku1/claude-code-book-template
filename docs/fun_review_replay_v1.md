## サマリー

「バズったー」は4ルート構造の設計として見ると、ルート間の連結素材は着実に仕込まれている。特に「ゆきわりそう」「カレン」というキーワードが複数ルートの素材カードに埋め込まれており、朔ルートの模型写真に喫茶店の看板が映り込む仕掛けは発見型フックとして機能する可能性がある。

一方で、重大な弱点が2点ある。第一に、**伏線型フックの大半が素材カードのめくり動作に依存しており**、プレイヤーが全カードをめくらない限り謎収束の条件（`mysteryClues.length >= 2`）を満たせない。素材カードをめくらなかった場合は `karen_mystery_choice` がスキップされ、最重要シーンが出現しない。第二に、**ルート間連結の情報が「知識」にとどまり「感情的再解釈」につながっていない**。誠司ルートの「花が好きな若い常連客」が花蓮である可能性が示唆されているが、花蓮ルートに戻ったときにその示唆が感情を揺さぶる形で回収されていない。

エンディング差別化については7種のエンドが存在し、感情の種類は概ね分かれているが、みどりルートの「自立エンド」と「継続エンド」が感触として近く、same feel問題の芽がある。花蓮ルートの「贖罪エンド（zange）」と「沈黙エンド（chinmoku）」は対比が鮮明で、両エンドとも2周目動機として強く機能する。

## 問題点・改善推奨（優先度順）

### 1. [重要度: 高] みどりルートが完全に孤立しており、リプレイ動機が生まれない

みどりはチュートリアル（タグ: チュートリアル）であるが、後の3ルートで「みどりへの言及」「みどりルートで知ったことが後で意味を変える」仕掛けが一切ない。4ルート構造の本来の意義（複数ルートで全容が見えてくる）がみどりでは機能していない。

**改善方向:** 朔または誠司の会話中に「みどりさんから紹介してもらった」または湊のモノローグ中に「みどりさんのときと似ている、でも違う」という一文を入れることで、みどりルートの体験が後ルートで活きる設計にする。あるいは、みどりの素材カードにmysteryFlagを1つ設定し、花蓮ルートの謎収束に貢献させる。

---

### 2. [重要度: 高] 謎収束の鍵（mysteryClues.length >= 2）が素材カードめくりに依存しており、到達が保証されない

`karen_mystery_choice` の condition は `mysteryClues.length >= 2` だが、mysteryFlagが設定されているカードは以下のみ:
- `s_desk`（朔: mysteryFlag: 'karen'）
- `s_model`（朔: mysteryFlag: 'yukiwarisou'）
- `sj_sign`（誠司: mysteryFlag: 'yukiwarisou'）
- `k_flower`（花蓮: mysteryFlag: 'yukiwarisou'）

花蓮ルートで `k_flower` をめくれば 'yukiwarisou' が1件加算されるが、それだけでは条件を満たさない。前のルート（朔または誠司）で対象カードをめくっていないと花蓮ルートの最重要シーンが出現しない。プレイ順序・カード選択によって謎収束が「見えない・見える」に分かれるため、謎が深まるのではなく「存在しない体験」になるリスクがある。

**改善方向:** `k_memo`（flashback3Trigger: true のカード）に mysteryFlag を追加するか、花蓮ルート内で条件なしに1件 mysteryClue が加算されるステップを設ける。あるいは `karen_mystery_choice` の condition を `mysteryClues.length >= 1` に緩和する。

---

### 3. [重要度: 高] `jiritu` と `jizoku` のsame feel問題（みどりルート内エンド差別化不足）

両エンドともに ending_piano、暗転演出、noReplyEffect: true という同じ演出構成を持ち、モノローグのトーンも「翻訳者の役割への省察」で近い。`jizoku` には clientMessage がなく、情報量が `jiritu` より少ない分だけ薄く感じられる。2周目でみどりルートを再プレイしても「あのエンドとは違う結末を見た」という差別化感が弱い。

**改善方向:** `jizoku` の BGM を 'ending_piano' から別トラック（または無音→小さな自然音）に変更する。あるいは `jizoku` に固有の clientMessage を追加し、「自分でまだ書けていないが続けている」ことを示す具体的なメッセージを付与する。

---

### 4. [重要度: 中] 誠司ルートの「花が好きな常連客」伏線が花蓮ルートで明示的に回収されていない

`seiji_regulars_detail` で「生け花をやってる若い常連客が急に来なくなった」が語られ、花蓮ルートの `k_flower` に「姉に見せたくて活けた」とあり、背景の「ゆきわりそう」看板が一致する。しかし花蓮ルート内で「誠司さんのお店に行ったことがある」または「あのお店、好きでした」という明示がない。プレイヤーが自力で気づく設計とも言えるが、気づかずに終わる可能性が高い。

**改善方向:** 花蓮ルートの会話中（`karen_intro_path` 付近）に「誠司さんのお店、一度だけ行ったことがあって」の一言を加えるか、`k_flower` のdescに「ゆきわりそうで活けた一作」と明記する。

---

### 5. [重要度: 中] 「到達エンド（toutatu）」の condition が kShinonoya=true を必須としているが、このフラグ入手の選択肢が目立たない

`saku_dilemma2_reply` の選択肢「…そのアカウント名、教えてもらえますか」は3択の中の一つで、かつ `egoPlus: true` が付いている（良い選択肢ではない印象）。しかし到達エンドはこのフラグが条件となっている。「怪しいフラグを取ることで良いエンドに近づく」という逆説的な設計が意図的なものかどうかが不明瞭であり、初見では回避される可能性が高い。

**改善方向:** 選択肢文言から「egoPlus」の印象を取り除くか、または到達エンド条件から kShinonoya を外し、selfScore のみで分岐させる。あるいは設計意図として「踏み込む選択が伏線を開く」と明示し、選択肢に微妙な引力を持たせる文言に改める。

---

### 6. [重要度: 低] 花蓮ルートの謎が「早解け」するリスク（仕様書パターンCの確認）

`karen_question`「昔、ライターさんでしたか？」は第1ターン後半に登場する。既に朔ルートのモノローグ（「3年前に削除されたはずのアカウント名」）と誠司ルートのモノローグ（「3年前に奪った居場所」）を経た後に花蓮ルートをプレイすると、花蓮の質問時点で「この人は何かを知っている」と確信できてしまう。仕様書の「花蓮の正体は第7シーン以降に確信」という設計と乖離している。ただし `k_memo` カードの `flashback3Trigger` がどのシーンで発火するかによって調整可能。

**改善方向:** 朔・誠司ルートのモノローグで「3年前」への言及を直接的にしすぎず、花蓮ルートで「確信」が生まれる展開のテンポを守ることを優先する。現状では示唆の密度がやや高い。

## 総評：2周目をやりたくなるか

花蓮ルートと朔ルートを繋ぐ伏線（「カレン」「k_shinonoya」「3年前」「ゆきわりそう」）は十分な密度で仕込まれており、この2ルートを往復するリプレイ動機は強い。特に到達エンドのモノローグ「それを言える立場に、私はない」は花蓮ルートの後に読み直すと全く異なる感情になる点で、感情再解釈型フックとして本作最高の仕掛けである。

ただし、謎収束の条件（`mysteryClues.length >= 2`）が素材カード選択に依存している構造上の問題と、みどりルートが4ルート連結に参加していない孤立問題が未解決のまま残っている。これらは「2周目で発見できる面白さ」ではなく「1周目の選択次第で体験が欠落する」問題であり、リプレイ価値以前の設計課題として対処が必要である。

総じて「朔・誠司・花蓮の3ルートをやれば2周目をやりたくなる」設計には届いているが、「みどりを含む4ルートの全容が見えてくる」という本来の設計意図には現状届いていない。優先修正3点（みどり孤立・謎収束保証・jiritu/jizoku差別化）を対処後に再評価を推奨する。
