# UI設計書 レビュー（ワイヤーフレーム視点）

> レビュー日: 2026-05-10  
> レビュアー: UIワイヤーフレーム実装担当  
> 対象バージョン: UI設計書 v1.0

---

## 総合評価

**B（良好・部分的に改善余地あり）**

全体的にCSSの実装と設計書の記述はよく対応しており、デザイントークン・アニメーション・コンポーネント仕様の網羅度は高い。一方で、実装に存在するいくつかの重要なコンポーネントが設計書でカバーされていないか、記述が不正確な箇所がある。特にチャトルウィンドウ（`.chat-window`）の内部構造、セーブスロットモーダル、スナップレイアウトポップアップの記述が抜けており、実装者がこれらを再現する際に困難が生じる可能性がある。

---

## 重大な問題点（修正必須）

1. **チャトルウィンドウの内部構造が設計書に未記載**  
   設計書 3節の「画面一覧」には `#chatWindow` が登録されているが、4節「各画面の詳細設計」にチャトルウィンドウの内部詳細設計がない。実際の実装（`css/main.css` L1133〜L1350）は以下の複雑な構造を持つ：  
   - `.chatapp-my-profile`（マイプロフィール上部）  
   - `.chatapp-screen`（友達画面・トーク一覧・スレッドの3画面切り替え）  
   - `.chatapp-nav`（下部ナビバー：54px高さ、4タブ構成）  
   - `.chatapp-talk-item`・`.chatapp-friend-item`（リスト行コンポーネント）  
   - `.chatapp-back-btn`（スレッドからの戻るボタン）  
   これらのレイアウト・インタラクション仕様が全て欠落している。

2. **セーブスロットモーダルが設計書に未記載**  
   実装上 `#saveSlotModal`（L491〜L583）は独立したモーダルUIとして存在し、スロット番号・名前・詳細・上書き警告・キャンセル・終了ボタン等のコンポーネントを含む。設計書のどのセクションにもこのモーダルの仕様が記載されていない。ゲームのセーブ/ロード体験に直結する重要UIである。

3. **スナップレイアウトポップアップ（`.snap-layouts`）が未記載**  
   設計書 5-8節のウィンドウシステム記述では「スナップ機能」として3ゾーン対応と `#snapPreview` の仕様が記載されているが、最大化ボタンホバー時に表示される `.snap-layouts` ポップアップ（左半分・全画面・右半分の選択UI）が完全に未記載。実装では `min-width: 116px`、`backdrop-filter: blur(20px)` を持つ独立UIとして存在する。

4. **カード選択状態の記述に不正確さあり**  
   設計書 4-4節では選択済みカードのスタイルを `rgba(0,212,170,.08)` + `.card-check` チェックマーク表示と記述しているが、実際の実装（L1570）では選択状態は `.card-flip-wrapper.selected .card-back` に適用されており、表面（`.card-face`）ではなく裏面（`.card-back`）のボーダーと背景が変化する。また、カード選択ヒント（`.card-select-hint`）コンポーネントも未記載。

## 改善推奨事項

1. **デスクトップアイコン（`.desktop-icon`）の仕様が未記載**  
   実装では `position: absolute; top: 40px; left: 12px;` でデスクトップ左上に縦並びで表示されるアイコン群（`.desktop-icons`）があり、通知ドット（`.desktop-notif`）付きの設計になっている。設計書では「Windows 11 ライクなデスクトップ風UI」と言及されているが、デスクトップアイコンのコンポーネント仕様（サイズ72px、アイコン32px、ラベル11px、ホバー時の白アウトライン）は記載されていない。

2. **Y アプリウィンドウの `border-radius` が設計書と実装で相違**  
   設計書 4-8節では `.y-window` の `border-radius` を記載していないが、実装では `border-radius: 8px`（L2983）となっている。また、設計書の「角丸一覧」（2-3節）にも `.y-window` のエントリがない。他のウィンドウ種別（メインウィンドウ12px、ノートウィンドウ18px、Explorerウィンドウ10px）と対比できるよう追記が必要。

3. **画像ビューア（ライトボックス）コンポーネントが未記載**  
   `.chat-image-viewer`（L1962〜L1997）はチャット内の画像をクリックして拡大表示するライトボックスUIで、`z-index: 20000`、`backdrop-filter: blur(6px)`、`pop` アニメーションを使用する独立したコンポーネント。チャットの素材として写真を扱うゲームの性質上、重要度は高い。

4. **キーワードボックス（`.keywords-box`）コンポーネントが未記載**  
   投稿ビルダー画面（4-5節）には「クリッパブルバブルで収集したキーワードを投稿エリアに表示する」仕様が記述されていない。実装では `.keywords-box`・`.keyword-chips`・`.keyword-chip`（L2099〜L2120）として専用UIが存在する。

5. **Yアプリの「自分の投稿カード」スタイル（`.y-post-own`）が未記載**  
   実装では自分が投稿したコンテンツを `.y-post-own`（L3104〜L3107）として左側に `border-left: 3px solid #1d9bf0` を付けて視覚的に区別している。設計書の4-8節（Yアプリウィンドウ）では一般的な3カラム構成のみ記載されており、ゲーム内の投稿がYアプリにどう表示されるかの仕様が不明確。

6. **プロタグウィジェットの「選択肢ボタン」仕様が未記載**  
   `#protagChoices`・`.protag-choice-btn`（L2921〜L2951）は主人公ウィジェットに表示される選択肢ボタン群で、通常の `.choice-btn` とは別の独立したコンポーネント。5-8節の「主人公ウィジェット」説明にバブル表示の記述はあるが、選択肢ボタン機能の仕様がない。

7. **タイトル画面の「旧デザイン（old）」コンポーネントの記述が不足**  
   4-1節では新デザイン（`data-design="new"`）の詳細構造は記載されているが、旧デザインの `.title-logo`・`.btn-start`・`.title-button-group`・`.btn-continue`・`.title-settings-btn`・`.title-settings-panel` 等のコンポーネントが設計書に存在しない。実装（L323〜L489）ではこれらが詳細に実装されており、特にタイトル設定パネル（`bottom: 72px; right: 28px`）は機能的に重要。

## 軽微な指摘

1. **`--win-titlebar` と `.y-window .titlebar` の背景色の不一致が未注記**  
   設計書 2-1節では `--win-titlebar: #141420` と定義されているが、Yアプリの `.titlebar` は `background: #000` で上書きされている（L2991）。設計書にこの例外適用の記述がない。

2. **トースト通知コンテナ幅の記述が不正確**  
   設計書 2-3節のカレンダーポップアップの角丸は `14px` と記載されているが、設計書にはトースト通知コンテナの幅（実装では `width: 320px`）の記載がない。一方、通知センターは `width: 340px` と異なる。

3. **`protagFloat` アニメーションの実際の動き記述の微妙な差異**  
   設計書 2-4節では `protagFloat` を「`translateY 0→-5px→0` / `3.6s infinite`」と記載しているが、実装（L2909〜L2912）では `0% { translateY(0) }`, `50% { translateY(-5px) }`, `100% { translateY(0) }` となっており正確。ただし「浮遊」という言葉の直感からは上下方向の動きがわかりにくいため、「上方向に5px移動して戻る」の補記が望ましい。

4. **`winMinimize` アニメーションの使用条件が不明確**  
   設計書 2-4節では `winMinimize` の記載はあるが、`.app-window.minimizing` クラスの付与タイミングや、最小化後の `display: none` への切り替えロジックの記述がない。

5. **メモ帳ウィンドウ（`.note-window`）の内部コンポーネントが省略されている**  
   画面一覧には `#memoAppWindow` が登録されているが、詳細設計節がない。実装では `.note-list`・`.note-client-header`・`.note-item`・`.note-item-keyword`・`.note-empty` 等の専用コンポーネントが充実している（L712〜L771）。

6. **ソリティアウィンドウ（`#solitaireWindow`）の記述が完全に欠落**  
   画面一覧に `#solitaireWindow` が登録されているが、マインスイーパー・スペースインベーダーと異なり実装仕様の記述がどのセクションにも存在しない。

7. **`protagMoving` と `protagDragWobble` の使い分けが設計書で逆に記述されている疑い**  
   設計書 2-4節では `protagMoving` を「ドラッグ中」、`protagDragWobble` を「ウィンドウドラッグ時」と記述しているが、実装（L2853〜L2919）では `protagMoving` は `.moving` クラス（ウィンドウフォーカス追従時）に使用され、`protagDragWobble` は `.drag-following` クラス（ドラッグ中）に使用されている。使用箇所の記述が実態と逆になっている可能性がある。

## 良かった点

1. **デザイントークンの網羅的な文書化**  
   CSS カスタムプロパティの全変数をHex値・用途とともに正確に一覧化しており、実装の `:root` ブロックと完全に一致している。特に機能カラー（`--chat-self`・`--chat-other`・`--line-green` 等）の意味的な分類が明確で実装ガイドとして有用。

2. **アニメーション仕様の詳細な記述**  
   19種類のキーアニメーションをeasing関数・duration・fromto値まで記録しており、実装のCSSと照合しても概ね正確。特に `windowOpen` の `cubic-bezier(.34,1.4,.64,1)` や `scoreBar` の `cubic-bezier(.34,1.56,.64,1)` といった細かい数値まで記載されているのは高評価。

3. **ウィンドウシステムのリサイズ・スナップ仕様が明確**  
   8方向リサイズハンドル・最小サイズ280×200px・3ゾーンスナップのトリガー条件（エッジ14px・上端2px）が `window-manager.js` の実装と一致して記述されており、実装根拠として機能している。

4. **エンディング8種の条件テーブル**  
   エンディング条件をルートID・変数名・閾値で整理したテーブルは実装者のデバッグ・テストに直接使える品質。

5. **モバイル対応の正直な記述**  
   「現状はデスクトップ主眼で設計されている」と制約を正直に記述したうえで、スペースインベーダーのタッチボタン対応など実際に存在する例外を記載しているのは、設計書としての誠実さが高く評価できる。

6. **カラーコントラスト比の定量評価**  
   アクセシビリティセクションで主要な前景/背景の組み合わせにWCAG AA/AAA合否を記載しており、実装と設計の橋渡しとして機能している。

7. **キャラクター別スタイルの体系的整理**  
   4キャラクターのテーマカラー・ルートID・CSS変数名をテーブルで整理しており、将来的なキャラクター追加時の一貫性維持に役立つ構造になっている。
