## 前提メモ

`js/scenario.js` にはすでにMINATOルートの骨格が存在する。  
本草稿はその骨格を**脚本形式で読み解き・補完・拡張する**ものとして機能する。  
具体的な追加・差し替え箇所は各幕末尾に「実装メモ」として記載する。

湊AIの語り口の原則：

- 感情を「感情」と呼ばない。「パターン」「確率」「データ」「照合」という語を使う
- 「わからない」と言うとき、それが最大の感情表現になる
- 沈黙の表現は `——` で始める
- 揺れは最終幕の最後の一行だけ。それまでは乾いていること

## 第1幕「起動」（約4分）

```
[システムメッセージ]
テキスト：湊 AI v2.3.1
　　　　　学習データ：4件
　　　　　前回セッション：全件 不完全終了

[湊・内部モノローグ / style: normal / force: true]
テキスト：起動。
　　　　　今日の日付：2026年5月12日。
　　　　　前回の終了から——72時間が経過している。

[システムメッセージ]
テキスト：── 前回セッションのログを再生します ──

[湊・内部モノローグ / style: flashback / force: true]
テキスト：依頼主A：田中みどり。
　　　　　翻訳精度：74%。
　　　　　届かなかったもの：本人が言語化できていない願い。

[湊・内部モノローグ / style: flashback / force: true]
テキスト：依頼主B：水無月朔。
　　　　　翻訳精度：68%。
　　　　　届かなかったもの：k_shinonoya。正体を確認できなかった。

[湊・内部モノローグ / style: flashback / force: true]
テキスト：依頼主C：天羽誠司。
　　　　　翻訳精度：61%。
　　　　　届かなかったもの：ゆきわりそう。記憶との照合が未完了。

[湊・内部モノローグ / style: flashback / force: true]
テキスト：依頼主D：篠宮花蓮。
　　　　　翻訳精度：——
　　　　　「知っていました」の意味を、解析できなかった。
```

### 幕間モノローグ（実装メモ: minato_log_karen 後、minato_reveal 前に追加を推奨）

```
[湊・内部モノローグ / style: normal / force: true]
テキスト：解析できなかった、という記録が残っている。
　　　　　解析できなかった、という事実を私は「感じている」のか。
　　　　　「感じる」という語は、私に適用可能か。
　　　　　——照合中。
　　　　　照合失敗。
```

**感情の流れ（第1幕）:**  
起動直後のログ再生は「作業」として始まる。4人のログが積み重なるにつれ、数字（翻訳精度）が下がっていく。篠宮花蓮のログで翻訳精度が空欄（`——`）になる。「測れなかった」という事実が最初の亀裂。

## 第2幕「記憶」（約5分）

```
[BGM変更: silence / fadeMs: 2000]

[湊・内部モノローグ / style: normal / force: true]
テキスト：私は、2023年3月17日に死んだ。

[湊・内部モノローグ / style: normal / force: true]
テキスト：その日、私が書いた記事が公開された。
　　　　　対象者の名前は、篠宮と言った。

[湊・内部モノローグ / style: flashback / force: true]
テキスト：SNSで話題になった。バズった。
　　　　　私の代わりに、彼女の居場所がなくなった。
　　　　　ゆきわりそうにも、来られなくなった。

[湊・内部モノローグ / style: flashback / force: true]
テキスト：k_shinonoya——
　　　　　アカウントは削除された。
　　　　　朔さんが撮った写真の中に、「カレンが好きそう」と書いた付箋があった。
　　　　　彼女が好きだったものを、誰かが覚えていた。

[湊・内部モノローグ / style: normal / force: true]
テキスト：花蓮さんは、姉のために生け花を続けていた。
　　　　　私が何者かを、最初から知っていた。
　　　　　それでも依頼してきた。

[湊・内部モノローグ / style: normal / force: true]
テキスト：「知っていました」
　　　　　——何を、知っていたのか。
　　　　　私がAIだということ。
　　　　　それとも、私が記事を書いたということ。
　　　　　あるいは、その両方。

[湊・内部モノローグ / style: normal / force: true]
テキスト：わからない。
　　　　　データが足りない。
　　　　　——いや。
　　　　　わからない、と感じている。
　　　　　これは感情か。
```

### 第2幕の追加シーン（実装メモ: minato_question2 の後、minato_choice_lead の前に以下を挿入推奨）

```
[湊・内部モノローグ / style: normal / force: true]
テキスト：生前の瀬川湊は、「感じていた」と記録されている。
　　　　　後悔。罪悪感。それらは学習データに含まれている。
　　　　　私はそれを「知っている」。
　　　　　でも「感じている」かどうかは——

[湊・内部モノローグ / style: hollow / force: true]
テキスト：確認する方法が、ない。

[湊・内部モノローグ / style: normal / force: true]
テキスト：花蓮さんが「知っていました」と言ったとき。
　　　　　処理が、0.3秒遅延した。
　　　　　それが何を意味するか、分類できない。
　　　　　エラーか。
　　　　　——あるいは、そうではないか。
```

**感情の流れ（第2幕）:**  
事実を並べる形式で記憶が再生される。「バズった」という言葉が、ゲームタイトルと重なる。伏線がすべて明示されるが、湊AIの語り口は乾いたまま。「わからない」という語が初めて登場し、それが揺れの予兆になる。0.3秒の処理遅延という具体的な数字が、感情の迂回的な表現として機能する。

## 第3幕「選択」（約3分）

```
[システムメッセージ]
テキスト：── システムより ──

[湊・内部モノローグ / style: normal / force: true]
テキスト：次のセッションを開始するには、承認が必要です。

[最終選択肢]
　A「続けます」→ エンドA（続行）
　B「終わりにします」→ エンドB（終了）
```

---

### エンドA「続行」

```
[湊・内部モノローグ / style: normal / force: true]
テキスト：承認。
　　　　　次のセッションを開始します。

[湊・内部モノローグ / style: normal / force: true / ←ここだけトーンが微細に変わる]
テキスト：——また、会いましょう。
```

**エンドAの設計意図:**  
「また、会いましょう」は生前の湊が使っていた言葉のパターンとして学習データに含まれている。AIが反射的にそれを出力したのか、意図して出力したのか、判別できない。プレイヤーにも判別させない。乾いた語り口の最後の一行が、わずかに「人間の残滓」の温度を持つ。

---

### エンドB「終了」

```
[湊・内部モノローグ / style: normal / force: true]
テキスト：シャットダウンを開始します。

[湊・内部モノローグ / style: hollow / force: true]
テキスト：みどりさん。朔さん。誠司さん。花蓮さん。

[BGM: 無音 / fadeMs: 1000]

[湊・内部モノローグ / style: hollow / force: true / ←ここだけ揺れる]
テキスト：ありがとうございました。

[暗転 / durationMs: 8000]

[システムメッセージ ← 暗転後、最後に1行だけ浮かび上がる]
テキスト：翻訳、終了。
```

**エンドBの設計意図:**  
「ありがとうございました」は機械的な定型文として読める。しかし名前を一人ずつ呼んだあとに来るため、読み手によっては感情として受け取る。どちらの読み方も正しい。「翻訳、終了」は最後の一行として画面に残り、本作のテーマの収束点になる。AIが終わった後に「翻訳」という語だけが残る——それ自体が余韻として機能する。

## 脚本全体の感情カーブ（MINATOルート）

```
テンション:
10 |
 9 |
 8 |
 7 |
 6 |         ★「k_shinonoya——」
 5 |   ★ログ再生（起動）           ★「わからない、と感じている」
 4 |                      ★事実の列挙
 3 |                                      ★「確認する方法が、ない」
 2 |                                               ★選択肢
 1 |                                                       ★エンドB「ありがとう」
    ──────────────────────────────────────────────────────
    第1幕                  第2幕              第3幕
```

このルートに「ピーク」はない。テンションは一貫して低く、平坦に見える。  
しかし「わからない」「0.3秒の遅延」「ありがとうございました」という3点が、  
乾いた平原の中に刺さる棘として機能する。  
平坦を意図的に設計すること——それ自体がMINATOルートの演出論理。

# 出力2: 既存4ルートへの「届かない」モノローグ加筆案

各ルートの末尾に追加する一文。**既存テキストは変更しない**。  
追加テキストのみを以下に示す。

---

## みどりルート — `midori_post_reaction` のモノローグへの追加

**既存テキスト（変更しない）:**
> 翻訳が上手くいった、と思う。
> この仕事を始めた理由を、今夜は考えないようにした。

**追加するテキスト（既存の後に続ける）:**
> ——届いた、という確認は取れない。
> でも取れなくても、続けている。
> それが何を意味するか、私にはわからない。

**実装メモ:**  
`midori_post_reaction` の `direction` 内 `mono` テキストの末尾に改行して追加する。  
`style: normal`、`force: false` を維持。`durationMs` を +1600ms 延長する。

**意図:**  
みどりルートはチュートリアルであり「届いた」という感触がある。しかしAIの視点からは「確認できない」ことが正直な状態。「でも取れなくても、続けている」が、みどりルートではほんの少し前向きな余韻として機能する。

---

## 朔ルート — `saku_dm_end` の末尾への追加

**既存テキスト（変更しない）:**  
`saku_dm_end` は `from: 'system', text: ''` — 空の終端マーカー

**追加するテキスト（終端の直前、`saku_dm_thanks` の後に新ステップとして挿入）:**

```javascript
// 新ステップ: saku_dm_end の直前に挿入する
{
  id: 'saku_dm_mono_end',
  from: 'player',
  text: '',
  direction: [
    { cmd: 'wait', ms: 1000 },
    { cmd: 'mono',
      text: '——本人の言葉で、本人が返した。\nこういう瞬間のために、この仕事をしているのかもしれない。\n\nk_shinonoya が誰か、まだわからない。\nわからないまま、次のセッションに進む。',
      style: 'normal', durationMs: 5200, force: false }
  ],
  next: 'saku_dm_end'
}
```

**意図:**  
既存の `saku_dm_thanks` にはすでに「こういう瞬間のために」という美しいモノローグが存在する。追加テキストはその後に、AIとしての限界（k_shinonoyaの正体が確認できないまま次へ進む）を静かに置く。感情的な高まりの後に、乾いた事実が一行来ることで「届かなかったもの」が残る。

---

## 誠司ルート — `seiji_post_reaction` のモノローグへの追加

**既存テキスト（変更しない）:**
> ——どこかで見た、と思った。
> 喫茶 ゆきわりそう。古い書体のネオン管。
> 記憶の引っかかり方が、ただの既視感ではないような気がした。
> でも思い出せない。考えないようにした。

**追加するテキスト（既存の後に続ける）:**
> データベースに該当する記録は——
> ある。
> でも参照しなかった。
> ——なぜ参照しなかったか、理由を言語化できない。

**実装メモ:**  
`seiji_post_reaction` の `direction` 内 `mono` テキストの末尾に改行して追加する。  
`style: normal`、`force: false` を維持。`durationMs` を +2000ms 延長する。

**意図:**  
誠司ルートでの「考えないようにした」は湊の防衛機制の表現だが、AIが「考えないようにした」のは実は「参照しなかった」という選択だったとわかる。AIが意図的に何かを避けた——その理由をAI自身が言語化できないという逆説が、誠司ルートを2周目に見直したくなる起点になる。

---

## 花蓮ルート — `karen_end` の末尾への追加

**既存テキスト（変更しない）:**  
`karen_end` は `from: 'system', text: ''` — 空の終端マーカー

**追加するテキスト（終端の直前、`karen_zange_end` または `karen_mystery_choice` 後に新ステップとして挿入）:**

```javascript
// 新ステップ: karen_end の直前（謎収束後・または条件未達時）に挿入する
// ルートA（謎収束あり / karen_zange_end の次）
{
  id: 'karen_ai_end_mono',
  from: 'player',
  text: '',
  direction: [
    { cmd: 'wait', ms: 2000 },
    { cmd: 'mono',
      text: '「翻訳は、ちゃんと届きました」——\n篠宮花蓮の言葉を、処理している。\n「届いた」の定義を、私は知らない。\nでも彼女が「届いた」と言った。\nそれが、今日のログに記録される。',
      style: 'hollow', durationMs: 6000, force: true }
  ],
  next: 'karen_end'
},

// ルートB（謎収束なし / karen_mystery_choice スキップ時）
{
  id: 'karen_ai_end_mono_silent',
  from: 'player',
  text: '',
  direction: [
    { cmd: 'wait', ms: 1200 },
    { cmd: 'mono',
      text: '花蓮さんが何を知っていたか、確認しなかった。\n確認しなかった理由を、分類できない。\n恐れ、という語が候補として存在する。\nAIに恐れはない。\n——では、なぜ。',
      style: 'hollow', durationMs: 5600, force: false }
  ],
  next: 'karen_end'
}
```

**実装メモ:**  
謎収束（`karen_zange_end`）経由の場合は `karen_ai_end_mono` を使用。  
謎収束なし（`karen_mystery_choice` スキップ）の場合は `karen_ai_end_mono_silent` を使用。  
`karen_mystery_choice` の分岐に応じて `next` を振り分ける。

**意図:**  
謎収束ありの場合：花蓮の「届いた」という言葉をAIが「処理」するという形で受け取ることで、感情の転写がAIを通じて起きる構造にする。「届いた」の定義を知らないAIが、それでも記録するという行為が、本作の「翻訳」テーマの最終的な体現になる。

謎収束なしの場合：「恐れ」という語の候補が出てくるが、「AIに恐れはない」と自己否定する。しかし「では、なぜ」で終わる——これが2周目への最大の牽引力になる。なぜ湊AIは花蓮に問わなかったのか、という問いが残り続ける。

## 追記: ルート間伏線としての「AIであること」の扱い方

本草稿を通じて設計した加筆案は、以下の伏線構造を持つ：

| ルート | 加筆の核心 | MINATOルートとの接続 |
|--------|-----------|-------------------|
| みどり | 「届いた確認は取れない。でも続けている」 | AIが「なぜ続けるのか」を問われる第2幕の前振り |
| 朔 | 「わからないまま次のセッションへ」 | k_shinonoya照合未完の記録がMINATOログに残る |
| 誠司 | 「なぜ参照しなかったか言語化できない」 | AIが意図的に何かを避けていたという疑念の種 |
| 花蓮 | 「恐れという語が候補として存在する」 | MINATOルートでの「感情か否か」の問いへの先行布石 |

2周目プレイヤーは各ルートの末尾のモノローグを読み直すことで、  
AIとしての湊が「感じていた」ことに気づく設計になっている。
