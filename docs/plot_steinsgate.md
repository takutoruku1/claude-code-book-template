## 1. 各ルートの伏線一覧

### みどりルート（伏線密度: 0〜10%）

#### 仕込む伏線: 1個

**伏線①「翻訳」という言葉**

場所: チャット終盤、投稿ビルダー完了後のみどりのリアクションシーン  
状況: みどりが投稿を見て「私の言葉に見える」と言う。湊の内部モノローグとして挿入。

> 湊（内部モノローグ）：「翻訳が上手くいった、と思う。その人の言葉として届く投稿——それが、この仕事の正解だ。」

**後から変わる意味:**  
花蓮ルートの贖罪エンドで「翻訳、ありがとうございました」という言葉が花蓮（または沈黙エンドでは差出人不明のメッセージとして）届く。「翻訳」という言葉が最初は仕事への誇りとして使われ、花蓮ルートで「贖罪と赦し」に変容する。

### 朔ルート（伏線密度: 30%）

#### 仕込む伏線: 3個

**伏線①「カレン」という名前の付箋（既存・強化）**

場所: 素材カード `s_desk`（作業台の全景）をめくった時  
状況: 現在の実装では `mysteryFlag: 'karen'` が蓄積される。これにセリフの肉付けを追加する。

> 湊（内部モノローグ）：「——カレン。」  
> 「付箋に書かれた名前が、目に入った瞬間、手が止まった。よくある名前だ。そう思おうとした。」

フラッシュバック演出: Phase 2（朔ルートの「残したい」発言後）のモノローグを次のように設計する。

> 湊（フラッシュバック、暗赤色）：「——残す。あの記事も、残っている。」

> 「2200文字。送信まで8分かかった。後悔するのに3日かかった。」

**後から変わる意味:**  
花蓮ルートで「朔さんから紹介されたんです」と花蓮が言う。朔の模型に「カレン」という名前が書いてあった理由が花蓮ルートで明かされる——朔は、姉を探し続けていた花蓮の友人だった。

**伏線②「誰かに届く」という言葉の二重性**

場所: 朔との選択肢「捨てなくていい。これが朔さんの視点です」を選んだ後のモノローグ  
状況: selfBonus +10 が入る選択肢の直後に湊の短いモノローグを挿入する。

> 湊（内部モノローグ）：「届く、という言葉を、私はいつも慎重に使う。」  
> 「届くのが良いことだとは限らない、と——一度だけ思い知ったから。」  
> 「でも今この瞬間は、届いてほしいと思った。朔さんの、この視点が。」

**後から変わる意味:**  
花蓮ルートで「言葉は届かなければよかった、と思った夜がありました」と花蓮が語る場面と対になる。「届くことの暴力性」が花蓮視点で語られることで、湊の過去が反転する。

---

**伏線③ 投稿後のDM通知「@k_shinonoya」**

場所: 朔ルートの反響フェーズ終了後、チャット再開（`saku_dilemma2`）直前  
状況: Yアプリ（Twitter/X風）のウィンドウにトースト通知が届く演出を追加する。

トースト通知テキスト: 「**Y** — 新しいDMが届きました」  
チャットに朔から「なんか、DM来た。@k_shinon... っていうアカウントから」と一言届く。

> 朔：「なんか来た。知らないアカウント。」  
> 朔：「...開けてもいい？」

ここでプレイヤーは返答する選択肢を持つ（または沈黙のまま次に進む）。朔が「開けてもいい？」と確認を求めることで、開封前の一呼吸をプレイヤーに与える。選択肢への回答に関わらず、朔はDMを開封してアカウント名を告げるステップに移行する（プレイヤーの「許可した」「止めようとして間に合わなかった」どちらの感情も受け取れる設計にする）。

選択肢（`saku_dm_notice` ステップとして追加）:
- 「…どうぞ」→ 朔がDMを開封する（明示的な許可）
- （何も言わない / 画面を眺める）→ 朔が自分で開封する（止めなかったという後悔）

どちらを選んでも次の湊のモノローグへ続く:

> 湊（内部モノローグ）：「k_shinonoya——」  
> 「3年前に削除されたはずのアカウント名と、最初の4文字が同じだった。」  
> 「偶然だ、と言い聞かせた。」

**後から変わる意味:**  
花蓮ルートで「姉は今も、名前だけ変えてネットを見ています」と語られる。`@k_shinonoya` と同じ文字列は姉の痕跡を探していた花蓮（または姉本人）からの接触だったことが示唆される。

## 3. 花蓮ルートのクライマックス構成

### 構成概要

```
花蓮ルート開始
  ↓
【導入】何も知らずに依頼を受ける（湊の平常心）
  ↓
【違和感の蓄積】素材カードに伏線が集中（k_flower: yukiwarisou / k_memo: flashback3）

  → 「篠宮」の二文字がチャットに現れる
  ↓
【謎収束】karen_mystery_choice（mysteryClues >= 2 の場合のみ）
  → 告白するか、沈黙するか
  ↓
【エンディング分岐】kidoku → zange / chinmoku
```

### 【フラッシュバック3の詳細設計】

**トリガー**: `k_memo`（依頼の理由メモ）をめくった後、`karenRevealDirections` の演出キュー完了後に発火（※「2500ms後」という固定タイマーは使わないこと。演出キューの合計所要時間が約12700ms超のため、固定タイムアウトで呼び出すと二重発火する）

`k_memo` の裏面テキスト（カード説明文）:

```
「言葉は怖くない」と証明したい。
姉に。
```

2500ms 後のフラッシュバック演出:

**Phase 1（既存）のテキストをここで画面に再掲する**（薄く、背景に）:
> 「深夜2時。送信ボタンを押した。」  
> 「@k_shinonoya のアカウントが削除されました」

その上に、現在の花蓮の素材カード `k_memo` が重なって表示される（2枚並列演出）。

**【2枚並列演出の意図・見せ方メモ（ビジュアル担当への引き継ぎ）】**  
この演出の意図は「3年前の罪と、今この瞬間の依頼が同一人物に連なっている」という湊の脳内の重ね合わせを視覚化することにある。「過去」と「現在」が同時に存在する瞬間の表現。  
見せ方の方針:
- 背景レイヤー（低透明度・約30%opacity）: Phase 1フラッシュバックのテキスト（「深夜2時。」「アカウントが削除されました」）を白テキストで薄くフェードイン
- 前景レイヤー: `k_memo` カードを画面中央に固定表示（通常カードと同じデザイン、ただし影を強調して「浮いている」感を出す）
- 重ねることで「カードの言葉が過去の行為の上に乗っている」という構図を作る
- 実装には絶対配置の `#flashback-overlay` レイヤーが必要。ビジュアル仕様書 `visual_spec.md` にこのレイヤーのCSS定義を追記すること（`z-index`・`position: fixed`・`opacity`トランジション）。

チャットに出現するテキスト:  
> 湊：「……篠宮」  
（`flashbackPhase = 3` セット）

---

### 【湊が気づく瞬間の設計】

フラッシュバック3発火後、チャットが再開する前に、主人公ウィジェットのモノローグバブルで以下が表示される:

> 湊（モノローグバブル、3秒間）：「——篠宮。」  
> 「カレンが好きそう、という名前。ゆきわりそう。既読のつかないメッセージ。」  
> 「頭の中で、点と点が線になっていく。」  
> 「線になった瞬間——吐き気がした。」

**【文体注意・演出担当への引き継ぎメモ】**  
このシーンの `overlay_text` 演出（`karenRevealDirections` 内）には、湊の身体感覚・繊細なモノローグ文体を使うこと。「俺が——追っていた。」のような男性的・能動的な断定口調は本作の世界観と乖離するため使用しない。使用すべき文体の基準は上記のモノローグバブルテキスト（「吐き気がした」「線になった瞬間」等の身体感覚表現）に準拠すること。オーバーレイテキストの例: 「頭の中で——全部、繋がっていく。」「吐き気がした。」

その後、花蓮からのチャットが来る（通常チャットとして）:

> 花蓮：「……湊さん？」  
> 花蓮：「顔色が悪いですよ、画面越しでも分かります。」  
> 花蓮：「大丈夫ですか。」

この「気遣い」が逆に湊の動揺を深める設計。

### 【花蓮が語る場面】karen_mystery_choice の直前チャット

## 5. 既存エンディングとのシュタゲ構造における位置づけ

### エンディング位置づけ一覧

| エンディング | シュタゲ構造における位置 | 伏線との関係 |
|------------|----------------------|------------|
| 自立エンド（みどり） | 「平和なタイムライン」。伏線はほぼ無い。ただし「翻訳」という言葉が種として残る。 | 後から「翻訳」の意味が変わることを予感させる最初の場所。 |
| 継続エンド（みどり） | 同上。より控えめな幸福。種すら気づかない。 | 伏線とほぼ無関係。初回プレイの安心感として機能する。 |
| ズレエンド（全ルート） | 「バズを取ることの空虚さ」を描く。どのルートのズレエンドでも「3年前も、こうだった。」が刺さる。 | 湊の罪の構造（バズのために誰かを傷つけた）を直接示す。花蓮ルートのズレエンドは最もグロテスク。 |
| 到達エンド（朔） | 伏線に「気づきかけた」ままで終わるエンド。kShinonoya を確認したが、それ以上追わなかった。 | 「届いた。」というラストが、後から「何に届いたのか」を問い返させる構造。 |

| 沈黙エンド（花蓮→） | 「翻訳、ありがとうございました。」という差出人不明のメッセージ。何も解決しない。でも何かが届いた。 | 全伏線が回収されたはずなのに、まだ何かが残っている感覚を残す。最も余韻の深いエンド。 |

### 贖罪エンド・沈黙エンドのシュタゲ的位置づけ（詳述）

**贖罪エンドと沈黙エンドは「同じ収束点の二面」として設計する。**

シュタインズゲートにおける「運命」のように、花蓮との対話は全ルートを通じて「起きることが決まっていた」収束点である。

- 贖罪エンドは「言葉を送ること」を選んだプレイヤーへの、花蓮からの赦しの保留。  
  告白した湊に対して、花蓮は「知っていました」と返す。これは赦しではなく、「承知の上で依頼した」という事実の確認だ。完全な赦しは描かれない。ただ「次に進める」という感覚だけが残る。

- 沈黙エンドは「言葉を送らなかった」プレイヤーへの、花蓮（または姉）からの静かな送信。  
  何も言わなかった湊のもとに「翻訳、ありがとうございました。」が届く。差出人は `——カ`。  
  カレンか、花蓮か。それとも、3年間送り続けた未読のメッセージが、ようやく届いたという比喩か。  
  答えは出ない。でも、それが——このゲームの本当のテーマだ。

---

### 既読エンドの「lastChoice」の設計意図

既存:
- 「……よかった」とだけ送る → egoScore +1 → 贖罪エンドへ
- （画面を閉じる）→ egoScore <= 0 → 沈黙エンドへ

**シュタゲ構造における意味:**  
この選択肢は「言葉を使う勇気があるか」という問いだ。  
湊は3年間、「言葉」を扱う仕事をしてきた。しかし自分の「言葉」を使うことをずっと避けてきた。  
最後の最後で、「たった2文字」を送れるかどうかが分岐点になる。

「よかった」という言葉は完璧ではない。謝罪でも告白でも説明でもない。  
でも——それが湊の「自分の言葉」だ。  
翻訳者が、初めて翻訳しないで言葉を送る瞬間。
